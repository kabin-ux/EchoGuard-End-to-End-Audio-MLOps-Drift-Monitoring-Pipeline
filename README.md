## ⚡ Quick Start

### Prerequisites

- Python 3.11+
- Docker Desktop (for the full stack)

### 1. Clone and install

```bash
git clone https://github.com/your-org/audio-genre-pipeline.git
cd audio-genre-pipeline

python -m venv .venv
# Windows
.venv\Scripts\activate
# macOS / Linux
source .venv/bin/activate

pip install -r requirements.txt
```

### 2. Generate ML Artifacts

This trains a Random Forest classifier and serializes `model.joblib` + `baseline.json`:

```bash
python scripts/generate_artifacts.py "data/genres_original"
```

Expected output:
```
08:00:00  INFO     Generating synthetic dataset (10 genres × 200 samples) …
08:00:01  INFO     Test accuracy: 98.00%
08:00:01  INFO     Model saved → app/artifacts/model.joblib  (512.3 KB)
08:00:01  INFO     Baseline saved → app/artifacts/baseline.json  (148.7 KB)
```

### 3. Run locally (dev mode)

```bash
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

Open [http://localhost:8000/docs](http://localhost:8000/docs) for the interactive Swagger UI.

### 4. Run the full Docker stack

```bash
docker-compose up --build
```

| Service    | URL                        | Credentials      |
|------------|----------------------------|------------------|
| FastAPI    | http://localhost:8000/docs | —                |
| Prometheus | http://localhost:9090      | —                |
| Grafana    | http://localhost:3000      | admin / mlops123 |

## 🔬 API Reference

### `POST /predict`

Upload an audio file to classify its genre.

```bash
curl -X POST http://localhost:8000/predict \
  -F "file=@my_song.wav"
```

**Response:**
```json
{
  "genre": "jazz",
  "confidence": 0.87,
  "latency_ms": 312.4,
  "drift": {

    "drift_fraction": 0.0625,
    "min_p_value": 0.142,
    "drifted_features": ["mfcc_mean_3"],
    "window_size": 30,
    "p_values": { "mfcc_mean_0": 0.621, "..." : "..." }
  },
  "features": {
    "mfcc_mean": [-122.3, 110.5, "..."],
    "spectral_centroid_mean": 1847.22,
    "chroma_mean": 0.4612
```

### `GET /metrics`

Returns Prometheus metrics in standard text exposition format.  
Scraped automatically by Prometheus every 5 seconds.

### `GET /health`

Liveness probe for Docker / Kubernetes health checks.

## 📊 Prometheus Metrics

| Metric Name                    | Type      | Description                                               |
|-------------------------------|-----------|-----------------------------------------------------------|
| `audio_predictions_total`      | Counter   | Total predictions, labeled by `genre` and `drift_detected`|
| `prediction_latency_seconds`   | Histogram | End-to-end latency (8 buckets from 50ms to 10s)          |
| `feature_drift_p_value`        | Gauge     | Minimum KS p-value across all features                    |
| `audio_upload_bytes`           | Histogram | Distribution of uploaded file sizes                       |
| `drift_window_size`            | Gauge     | Rolling window depth for drift calculation                |

## 🧠 Feature Engineering

Extracted from each audio file using **Librosa** (30 features total):

| Feature                | Statistic | Dimensions |
|------------------------|-----------|------------|
| MFCC                   | mean + std | 26 (13×2) |
| Spectral Centroid      | mean + std | 2          |
| Chroma STFT            | mean + std | 2          |
| **Total**              |           | **30**     |

---

## 📡 Data Drift Detection

The KS engine runs a **two-sample Kolmogorov-Smirnov test** (via `scipy.stats.ks_2samp`) for every feature dimension:

1. **Rolling window**: last 30 predictions are collected.
2. **Per-feature test**: each feature's distribution is compared against `baseline.json`.
3. **Alert threshold**: drift is flagged if ≥ 25% of features have p-value < 0.05.
4. **Prometheus**: `feature_drift_p_value` gauge exposes the worst-case p-value.

## 🚀 CI/CD Pipeline

GitHub Actions workflow (`.github/workflows/ci.yml`) runs on every push:

1. **Lint** — Ruff static analysis
2. **Artifacts** — `python scripts/generate_artifacts.py`
3. **Test** — `pytest` with XML coverage report
4. **Docker Build** — Builds the image and verifies `/health` returns 200
